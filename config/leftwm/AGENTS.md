# AGENTS.md

## Scope
Contributor guidance for the LeftWM config in this directory only.

## Structure
- `config.ron` is the main LeftWM configuration and primary source of truth for behavior.
- `themes.toml` tracks theme definitions and selection metadata.
- `themes/` contains maintained user theme assets; `themes/current/` is the active theme payload in this tree.
- `up` and `down` exist at the top level, but the main maintained config surfaces are `config.ron`, `themes.toml`, and `themes/`.

## Sources of truth
- Use `config.ron` for keybinds, layouts, tags, scratchpads, focus behavior, and core WM settings.
- Use `themes.toml` for registered theme metadata and which theme is marked current.
- Use `themes/` for theme-local assets and theme-specific configuration files.

## How to change config safely
- Edit `config.ron` for window manager behavior changes rather than burying behavior in theme assets.
- Edit `themes.toml` when changing theme selection/registration metadata.
- Edit files under `themes/` when changing the actual maintained theme contents.
- Keep behavior/config changes separate from appearance/theme changes when possible.

## What not to touch casually
- Do not casually rewrite large portions of `config.ron`; it is the load-bearing LeftWM config.
- Do not assume everything in `themes.toml` is local authoring; it includes theme repository metadata, so change entries deliberately.
- Do not confuse embedded theme contents under `themes/` with disposable runtime state; this is maintained user config in this repo.
- Do not move theme-specific settings into `config.ron` unless they truly belong to core WM behavior.

## Practical contributor notes
- `config.ron` is the first place to inspect for user-intended behavior.
- `themes.toml` and `themes/` together define the theme layer.
- Treat `themes/current/` as maintained configuration/assets for the currently selected theme, not as an autogenerated cache.
